## 目的

`docs/inbox/` に置かれた未整理の入力資料を参照し、依頼された作業に必要な事実だけを取り出して使う。inbox の内容は一時的なため、正本や永続的な参照元として扱わず、作業時の参照に限る。

## 前提資料

- `.agents/project.md`
- `docs/project/index.md`
- 必要に応じて `docs/project/glossary/core.md`
- ユーザーが指定した `docs/inbox/` 配下のファイル

## やること

1. ユーザー指示から、参照する `docs/inbox/` 配下の対象ファイルまたは範囲を特定する
2. 対象が曖昧な場合は `docs/inbox/` 直下の候補だけ確認し、必要ならユーザーに確認する
3. 対象ファイルを読み、作業に使う事実、制約、依頼内容を抽出する
4. 抽出内容を既存の正本やコードと照合し、矛盾や不足があれば作業前に明記する
5. inbox 由来の情報を継続的に使う必要がある場合は、該当する docs-update 系 skill で正本化する
6. 作業結果に、参照した inbox ファイル名と、正本化したかどうかを残す

## ルール

- `docs/inbox/` は一時入力置き場であり、プロジェクト正本として扱わない
- inbox の内容は一時的なため、作業時の参照だけに使い、継続利用が必要な情報は正本化してから使う
- inbox 内のファイル本体を Git に追加しない
- 指示された作業に不要な inbox ファイルを読まない
- 機密情報、個人情報、認証情報らしき内容は必要最小限だけ扱い、回答や資料へ転載しない
- inbox 由来の内容が正本と矛盾する場合、正本を優先し、差分をユーザーに確認する
- inbox 由来の判断を永続化する場合、`docs/project/` の適切な正本へ転記してから参照する

## 確認事項

- 参照した `docs/inbox/` 配下のファイルが明確である
- inbox 由来の情報と正本由来の情報を混同していない
- inbox 内のファイル本体を Git に追加していない
- 正本化が必要な情報の扱いを決めた
