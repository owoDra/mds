# Core Glossary

このファイルは、プロジェクト全体で必読となる共通用語を置きます。

## mds

このプロジェクトの名称。正式名称は Markdown Source。
Markdown を実装正本として扱い、複数言語向け生成・検証・補助ツールを提供するこのプロジェクト全体、またはその中核ツール群。

## mds-cli

`mds` コマンド本体。init、build、lint、typecheck、test、doctor、package sync、self-update の入口。

## mds-core

`mds` の中核 library。config、descriptor、package 発見、Markdown 解析、生成計画、quality 実行、診断を担う。

## mds-lsp

Markdown 実装文書向けの Language Server Protocol 実装。診断、補完、hover、navigation、source map 連携を担う。

## mds file

`mds` が authoring 対象とする Markdown 文書。通常は `.mds/source/**/*.md` または `.mds/test/**/*.md` 配下に置かれる。

## impl md

`.mds/source/**/*.md` に配置される実装コードの正本となる Markdown 文書。言語ラベル付き code fence を含み、生成元になる。

## test md

`.mds/test/**/*.md` に配置されるテストコードの正本となる Markdown 文書。生成時は test output 側へ写像される。

## package

`mds.config.toml` と package manager metadata を持つ処理単位。`mds` は package ごとに config 読み込み、文書探索、生成、quality 実行を行う。

## config

package に配置される `mds.config.toml` ファイル。

## descriptor

ツールチェーンの出力を mds file 向けに出力を変換するための設定ファイル。ツールごとに定義。

## source map

Markdown 上の code block と生成後ファイル位置を対応付ける写像。LSP の diagnostics や navigation の逆引きに使う。

## generated file

`mds` が生成またはコピーした派生ファイル。`Generated by mds` header を持つ出力と asset copy を含む。

## Plan

実装計画やロードマップの正本。`docs/project/plan/` に置き、検証可能な phase と実作業単位の plan task に分解する。

## Plan Task

`docs/project/plan/<plan-slug>/phase-*/task-*.md` に置く、実装計画上の作業単位。1 Markdown ファイルにつき 1 task とし、前提条件、作業内容、完了条件、検証方法、依存関係、成果物を記載する。

## 作業ログ Task

`.agents/tasks/task-*.md` に置く、AI エージェントの作業ログまたは文脈キャッシュ。plan task の正本ではなく、作業中の判断、検証結果、未完了事項を引き継ぐために使う。
