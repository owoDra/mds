# 生成コード出力

## 概要

mds は source md の `Source` と `Types`、test md の `Test` に書かれた実コードブロックを規約に従って派生ファイルへ出力する。

## 関連要求

- `../../requirements/REQ-generation-code-output-rules.md`

## 入力

- `.mds/source/**/*.{lang-ext}.md`
- `.mds/test/**/*.md`
- `mds.config.toml` の output root
- language descriptor の file rule
- `.mds/manifest.toml`

## 出力

- Source ファイル
- Types ファイル
- Test ファイル
- `.mds/manifest.toml`

## 挙動

- Source は source md の `Source` のコードブロックを language descriptor の `files.source` 規則に従って生成する。
- Types は source md の `Types` のコードブロックを language descriptor の `files.types` 規則に従い、原則として別ファイルへ生成する。
- Test は test md の `Test` のコードブロックを language descriptor の `files.test` 規則に従って生成する。
- core は `source_root`、`types_root`、`test_root` の概念を持ち、具体 default は language descriptor が言語慣習に合わせる。
- source md と test md は `.mds/source`、`.mds/test` からの相対パスまたは logical module id を保って `source_root`、`types_root`、`test_root` へ生成する。
- `Types`、`Source`、`Test` 内のコードブロックは出現順に連結する。
- 複数コードブロック連結時は、各コードブロック末尾を整え、ブロック間を LF 1 行で連結し、出力末尾は LF 1 行にする。
- コードブロックの間には説明文や補足見出しを置ける。
- language descriptor は言語ごとの Source / Types / Test の既定 file rule と、追加生成物が必要な場合の更新規則を提供する。
- TypeScript / Python / Rust の既定 file rule と言語固有の追加生成規則は、各 language adapter の subproject 固有 spec で定義する。
- mds 管理 marker は、生成ファイル先頭の adapter 別コメント header と package 内 manifest を併用する。
- 生成ファイル header は adapter 別コメントで `Generated by mds. Do not edit. Source: <path>. Source-Hash: <sha256>.` を入れる。
- 生成 manifest は package root の `.mds/manifest.toml` とする。
- `.mds/manifest.toml` は source 単位に `path`、`adapter`、`hash` と outputs の `kind`、`path`、`hash` を持つ。
- manifest hash は、source は正規化済み生成入力、output は生成後内容を SHA-256 で計算する。

## 状態遷移 / 不変条件

- 自由な `file=` 指定は採用しない。
- authoring root は `.mds/source` と `.mds/test` に固定し、生成 root とは分離する。
- 生成コードは派生物であり、正本は Markdown とする。
- `mds build` は mds 管理 marker を持つ生成物だけ上書きする。
- `.mds/source/overview.md` の managed dependency snapshot が stale な場合、`mds build` は生成前に error で停止する。
- 対応元が消えた stale 生成物は warning に留め、自動削除しない。
- 1 source md から複数機能の Source を生成しない。
- generator は設計説明から実装コードを合成せず、Markdown 内のコードブロックを生成元とする。

## エラー / 例外

- descriptor が対象言語の file rule を解決できない場合は生成エラーにする。
- 出力先が対象 package の範囲外になる場合は reject する。
- 出力先に既存ファイルがあり、mds 管理 marker がない場合は生成エラーにする。

## 横断ルール

- output root は package 設定解決後の値に従う。
- 生成後の import / use / require は `Uses` から adapter が生成する。
- `mds build --dry-run` は書き込みを行わず、生成計画に加えて git 互換寄りの unified diff を表示する。

## 検証観点

- 各 language adapter の代表 fixture から期待ファイル名を導出できることを確認する。
- 出力コードと Markdown 内コードブロック由来の期待出力を比較する。
- 自由な `file=` 指定が使えないことを確認する。
- TypeScript / Python / Rust fixture で `.mds/manifest.toml`、生成 header、adapter 固有の追加生成物を確認する。
- `mds build --dry-run` が書き込みなしで生成計画と unified diff を表示することを確認する。
