# Markdown 正本

## 目標

mds は Markdown を設計、仕様、実装、テストを追跡できる正本として扱い、Markdown 内に書かれた実装レベルのコードから生成コードを派生物として出力できること。

## 根拠

人間と AI エージェントが同じ Markdown を読んで、実装、修正、検証を再現できる状態を作るため。Markdown は AI にコード生成を依頼するための設計メモではなく、実コードを含む一次情報である必要がある。

## 対象範囲

- 設計、仕様、実装、テストの根拠と実コードを Markdown に置くこと
- `.ts`、`.py`、`.rs` などの生成コードを Markdown 内のコードブロックとメタ情報から導出できること
- 設計とコードを分離しすぎず、同じ正本から追跡できること

## 対象外

- 生成コードを正本として手動管理する運用
- 設計説明だけをもとに AI が実装コードを推測生成する運用
- Markdown 以外の形式を正本にすること
- 任意の自由形式 Markdown を無制限に解釈すること

## 成功指標

- 代表的な実装 md 内の `Types`、`Source`、`Test` コードブロックから生成コードとテストが再生成できる
- 人間が Markdown だけを読んで対象機能の目的、契約、依存、公開面、検証観点を把握できる
- AI エージェントが Markdown 正本を根拠に修正対象と検証対象を特定できる

## 制約 / 品質条件

- 自由度より規約を優先する
- 正本と生成物の責務を混同しない
- AI は Markdown 正本を編集・検証する補助者であり、設計文から暗黙に実装を補完する生成元ではない
