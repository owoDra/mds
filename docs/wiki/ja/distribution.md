# 配布方針

このページでは、mds の配布方針を説明します。

## 基本方針

mds は、Rust で実装した中核処理とコマンドを中心に、複数の環境から利用できる形を目指します。

利用者が使っている言語環境に合わせて導入できるように、Cargo、npm、Python パッケージ、ネイティブ実行ファイルでの配布を対象にします。

## 配布対象

| 配布経路 | 役割 |
| --- | --- |
| Cargo | Rust 環境から mds を導入する経路です。 |
| npm | Node.js 環境から mds を呼び出す経路です。 |
| Python パッケージ | Python 環境から mds を呼び出す経路です。 |
| ネイティブ実行ファイル | 言語環境に依存せず mds コマンドを実行する経路です。 |

## 中核処理

中核処理は Rust で実装します。

Rust は、Markdown の解析、設定解決、パッケージ検出、生成計画、ファイル生成、診断などを担当します。

## npm からの利用

npm パッケージは、ネイティブ実行ファイルを呼び出すための入口として扱います。

npm 側で mds の意味体系を変えるのではなく、同じ mds コマンドを Node.js 環境から呼び出せるようにします。

## Python からの利用

Python パッケージも、ネイティブ実行ファイルを呼び出すための入口として扱います。

Python 側で別の意味体系を作るのではなく、同じ mds コマンドを Python 環境から呼び出せるようにします。

## 公開前の品質確認

公開前には、配布物の品質を確認します。

確認対象には、次のようなものがあります。

- 配布物の存在
- チェックサム
- 署名
- ソフトウェア部品表
- 来歴情報
- インストール後の簡易動作確認
- 呼び出し用パッケージからネイティブ実行ファイルを起動できること

これらの確認は、公開後に導入できない成果物を配布しないためのものです。

## 現在の状態

この配布方針は実装と公開準備の対象です。

利用できる導入方法は、リリースの進行に応じて変わります。現時点で確実に試す場合は、リポジトリから Rust のコマンドとして実行してください。
