# モノレポでの使い方

このページでは、複数のパッケージを含むリポジトリで mds を使う考え方を説明します。

## 基本方針

mds は、パッケージ単位で対象を判断します。

ひとつのリポジトリに TypeScript、Python、Rust のパッケージが混在していても、それぞれのパッケージで mds を有効にするかどうかを決められます。

## 対象パッケージの判定

mds は、主に次の情報から対象パッケージを判断します。

| 情報 | 役割 |
| --- | --- |
| `mds.config.toml` | mds が有効かどうかを判断します。 |
| `package.md` | パッケージ単位の説明と同期対象を確認します。 |
| `package.json` | Node.js パッケージ情報を読み取ります。 |
| `pyproject.toml` | Python パッケージ情報を読み取ります。 |
| `Cargo.toml` | Rust パッケージ情報を読み取ります。 |

対象外のパッケージは、勝手に生成や書き換えの対象にしません。

## `--package` の指定

特定のパッケージだけを対象にする場合は、`--package` を指定します。

```bash
mds check --package packages/example
```

`--package` を省略した場合は、現在のディレクトリ以下から mds が有効なパッケージを探します。

## 複数言語の扱い

ひとつのリポジトリに複数の言語があっても、言語ごとの差分は言語アダプターが担当します。

| 言語 | 主な判定対象 |
| --- | --- |
| TypeScript | `*.ts.md`、`package.json` |
| Python | `*.py.md`、`pyproject.toml` |
| Rust | `*.rs.md`、`Cargo.toml` |

同じリポジトリ内でも、パッケージごとに有効な言語を変えられます。

## 生成先の安全性

mds は、生成先が対象パッケージの外へ出る場合はエラーにします。

また、mds 管理ヘッダーがない既存ファイルを上書きしません。これにより、同じリポジトリ内の他のパッケージや手書きファイルを保護します。

## モノレポでの推奨運用

- パッケージごとに `mds.config.toml` を置きます。
- パッケージごとに `package.md` を用意します。
- 実装 Markdown は既定の `src-md` に分離します。
- 生成コードと手書きコードの責務を混ぜないようにします。
- 初回は `mds build --dry-run` で生成予定を確認します。
- 継続的な検査では `mds check` を先に実行します。
