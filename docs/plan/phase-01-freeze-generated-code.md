# Phase 01: 生成済みコードの正本化

## 前提

- Phase 00 により、first-party self-hosting 停止が正本上の方針になっている。
- 現在の `mds/core/src`、`mds/cli/src`、`mds/lsp/src`、`mds/**/tests` を保持する実装として扱う。
- 生成元 `.mds` と生成後 `.rs` が並存していても、この phase 以降の編集対象は `.rs` と `.ts` 側である。

## やること

- `mds/core/src/**/*.rs`、`mds/cli/src/**/*.rs`、`mds/lsp/src/**/*.rs` を canonical source として固定する。
- first-party tests を canonical tests として固定する。
- `Generated by mds. Do not edit.` のような generated header を削除するか、必要な通常コメントへ置き換える。
- first-party code を変更するときに `.mds/source` を編集する運用を停止する。
- Cargo workspace が direct source tree を参照していることを確認する。
- `mds build` を first-party source 更新手順として呼ぶ workflow、docs、scripts を削除または無効化する。

## 完了条件

- first-party source files が直接編集可能な通常 source として扱われる。
- generated header が canonical source に対して「編集禁止」と表示しない。
- `cargo check --workspace` が direct source tree を対象に動く。
- source of truth が `.mds` と `.rs` に二重化されていないという方針が明確である。

## 注意事項

- generated header の一括削除は広範囲差分になる。機械的変更としてまとめ、挙動変更と混ぜない。
- generated code の formatting 差分が出る可能性があるため、header 削除後に `cargo fmt` の扱いを確認する。
- `.mds` directory は Phase 02 まで残っていてよい。Phase 01 の目的は canonical source の切り替えであり、物理削除ではない。
- 既存 tests が self-hosted assumptions を持つ場合、Phase 02 で削除・置換する。Phase 01 では必要最小限の修正に留める。
